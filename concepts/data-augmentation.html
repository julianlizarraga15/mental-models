Main idea

🖼️

Change the image

rotation, crop, blur, brightness

=

Keep the answer

the object / region is still the same

🧠

Teach invariance

ignore irrelevant visual changes

Visual model

Imagefixture labeled WC
↴
Flip / rotate / cropblur / noise / brightness
↳
Still WCsame label, same box adjusted

Good vs bad

✓

Good augmentation

changes pixels without changing meaning

✕

Bad augmentation

changes meaning but keeps the old label

⚖️

Rule

augment only like production can vary

For blueprints / takeoffs

👍

Usually helpful

small rotation, blur, contrast, DPI, rasterization artifacts, small crops, line thickness

⚠️

Risky

horizontal flips, strong rotation, cropping out the tag, changing meaningful colors

▦

Detector detail

when the image moves, boxes must move too

When it helps

Use augmentation when:

real-world variation is larger than training-data variation.
